Payload Diff {{ trace_id }}

左右並排比較這次 LLM 呼叫的兩個關鍵 stage 的原始 JSON： 左邊 context 是 agent 內部以為要送的內容， 右邊 before_provider_payload 是經過 pi-ai provider adapter 改寫後即將送 HTTP 給上游 LLM 的內容。

怎麼看這個並排比較？

顏色含意：

  • 沒底色：兩邊一模一樣
  • 綠色：只在右邊有（adapter 自己加的）
  • 紅色：只在左邊有（adapter 拿掉的）
  • 黃色：兩邊都有但內容不一樣
  • 灰色：佔位（讓兩邊行數對齊）

常見模式：

  • 左邊有 "timestamp": ...、右邊沒有 → adapter 把 agent 自己的 metadata 拿掉了（model 不需要看）
  • 右邊有 "stream": true、左邊沒有 → adapter 自動開了 streaming
  • 左邊 "role": "toolResult"、右邊 "role": "tool" → OpenAI-compatible API 用 tool 不用 toolResult
  • 右邊出現 "reasoning": "..."、左邊沒有 → adapter 把 assistant 內容裡的 <think> block 抽出來變成獨立欄位
