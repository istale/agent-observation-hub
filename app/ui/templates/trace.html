怎麼看這個頁面？

Agent Events ⓘ {% if session_id %}session {{ session_id }}{% endif %}

下方按時間順序顯示 Pi 從使用者輸入到 model 回應之間經歷的每個 stage。每一個 stage 都是 Pi 內部一個函式邊界的快照。色塊代表事件分類： 使用者輸入 資源載入 送給模型前 tool 呼叫 tool 結果

各 stage 的意義（第一次來必看）
  • before_agent_start：使用者按下送出的瞬間。raw_user_text 是使用者真實打的字；如果你 pin 了 constraint，effective_text 是 Pi 真正會處理的字。
  • resource_loaded：Pi 啟動時掃描的 skills（額外能力包）、prompt templates（可重用的指令模板）、AGENTS.md（專案說明文件）清單。
  • before_provider_request：Pi 準備呼叫 model 的一些 metadata（model id、message 數量、tool 數量、timeout）。
  • context：Pi 內部 message 陣列、tool schema。這是「agent 自己以為要送什麼」的快照。
  • before_provider_payload：經過 pi-ai provider adapter 改寫後、即將送出 HTTP 之前的完整 payload。這是「模型實際看到什麼」的快照。跟 context 比較就能看到 adapter 改了什麼。
  • tool_call：model 決定要用 tool 了。args 是 model 給 tool 的參數。
  • tool_result：tool 跑完了。包含 stdout / 是否失敗 / 花了幾 ms。

seq 是同一個 trace 內的序號（從 1 開始遞增），跟 timestamp 一起用可以確認順序。

📜 對話流程 Turn Timeline ⓘ

下方按時間順序顯示 Pi 處理使用者 prompt 的每個 stage，並依「round」分群——一個 round 是一次 model 呼叫或一次 tool 執行。 這次共有 {{ model_call_total }} 次 model 呼叫。

「Model Response」在哪裡看？為什麼有時候有兩次 Model Call？

Model 沒有獨立的 response event。Model 回完之後，它的回應有兩種去處：

  • 📨 直接給使用者的最終回答：顯示在頁面下方的「LLM Calls」區塊的 Assistant 欄位。
  • 🔧 請 agent 幫忙呼叫 tool：顯示在這個 round 之後的「Tool Execution」事件。

為什麼一次使用者輸入會觸發多次 Model Call？

典型的 tool 使用流程：

  1. Model Call #1：model 看了 user 的問題後，決定「我需要先用 bash 看一下檔案」→ 回傳 tool_call
  2. Tool Execution：Pi 執行 bash → 拿到結果
  3. Model Call #2：Pi 把 tool 結果塞進 context 再問 model 一次 → model 這次直接回答 user

每次 Model Call 都會有自己一組 before_provider_request + context + before_provider_payload。第二次的 context 會比第一次多一筆 toolResult 訊息——你點開比對就能看到。

典型 timeline 長這樣（對照下方 Turn Timeline 看）：

🛠 Session 啟動設定
└ resource_loaded                                                    ← Pi 啟動時掃描 skills / templates / AGENTS.md

📜 對話流程 Turn Timeline
👤 使用者送出 prompt ↓
└ before_agent_start                                                 ← 收到 user 輸入

🤖 Model Call #1
└ before_provider_request                                            ← Pi 準備呼叫 model
└ context           ← model 這時看到：使用者問題
└ before_provider_payload ← 送 HTTP 出去，model 回應：「我要用 bash」

🔧 Tool Execution
   ↳ tool_call      ← model 要求的 args（例如 {"command": "echo hi"}）
   ↳ tool_result    ← bash 跑完的 stdout

🤖 Model Call #2 (最後一次)
└ before_provider_request
└ context           ← model 這時看到：使用者問題 + tool 結果
└ before_provider_payload ← 送 HTTP 出去，model 給最終回答
