列出這個 session 的所有 messages（從 Pi 的 session JSONL 讀來）。 原始 JSONL 不會被修改——你的標記與筆記存在 hub 的 SQLite，作為一層 overlay。

怎麼用這頁（第一次來必看）

把它想成「桌面整理」：

  • active 預設。訊息正常參與 conversation。
  • background 不是當下重點，但保留——「收進抽屜」。UI 會把它變淡。
  • stale 已過時，但留著供 audit。UI 加刪除線。
  • hidden 預設折疊，點開才看得到內容。「移到角落」。

標記不會改原 JSONL 檔。之後若 Pi 端整合（未來 Cut），它會在送 model 前用 overlay 來降權標 background/stale 的訊息。目前先當「視覺整理工具」用。

Note：對任何訊息加一段你自己的註解（例如「這次決定後面都還在用」「這個 tool 用法是錯的」）。同樣不會改原訊息，只是提醒未來的你。
